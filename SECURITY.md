# Security Policy

## Supported Versions

Only the latest release is actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < latest| :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not open a public issue**.

Instead:

1. Go to the repository on GitHub.
2. Click **Security → Report a vulnerability** (Private vulnerability reporting).
3. Describe the issue in as much detail as possible.
4. Include steps to reproduce if applicable.

I will respond as soon as possible and work with you to assess and address the issue before any public disclosure.
